# Issue 15: デザイン修正（アイコンとレイアウト）

## 概要
既存のUI要素を改善し、視覚的な明確さとユーザーエクスペリエンスを向上させる。特に、削除ボタンをアイコンに変更し、タスク詳細の表示問題を解決する。

## 関連Issue
- Issue #3: マテリアルデザイン、ガラスのような透明感、白を基調とした色味、オブジェクトのセンタリング
- Issue #11: デザイン修正（初期）
- Issue #13/14: デザイン修正（追加）

## 実装計画

### フェーズ1: Deleteボタンのアイコン化
1.  **ゴミ箱アイコンの選定と導入:**
    *   適切なゴミ箱アイコン（例: Font Awesome, Material Icons などのSVGまたはコンポーネント）を選定する。
    *   プロジェクトにアイコンライブラリが導入されていない場合、軽量なSVGアイコンを直接使用するか、必要に応じてライブラリを導入する。
2.  **`src/components/TaskItem.tsx` の修正:**
    *   既存の「Delete」ボタンを、選定したゴミ箱アイコンに置き換える。
    *   アイコンのサイズ、色、配置を調整し、既存のデザインと調和させる。
    *   アイコンがクリック可能であることを視覚的に示すためのスタイル（例: ホバー時の変化）を追加する。

### フェーズ2: タスク詳細（content）の表示修正
1.  **`src/components/TaskItem.tsx` の修正:**
    *   現在、`title` をクリックすると `content` の表示/非表示が切り替わるが、これが機能しない問題を解決する。
    *   `onClick={() => todo.content && setIsOpen(!isOpen)}` のロジックと、`isOpen && todo.content` の条件を再確認し、デバッグする。
    *   `content` が表示される領域のスタイルを調整し、読みやすさを確保する。

### フェーズ3: レイアウト調整（垂直方向の配置）
1.  **`app/app.css` または関連するCSSファイルの修正:**
    *   「valignはtopから少し間を空けたくらいにして、中央からは上にしてください」という指示に基づき、主要なコンポーネント（特にタスクリストやモーダル）の垂直方向の配置を調整する。
    *   `align-items: flex-start` や `padding-top`、`margin-top` などのCSSプロパティを適切に利用し、デザインの意図を反映させる。
    *   特に、`main-container` や `todo-list-container` の垂直方向の配置を見直す。

### フェーズ4: 全体的な調整とテスト
1.  **CSSの整理と最適化:**
    *   追加されたスタイルが既存のスタイルと競合しないか確認し、必要に応じて `app/app.css` 内で整理・統合する。
    *   インラインスタイルは可能な限り `app/app.css` に移動する。
2.  **Vitestによる単体テストの実行:**
    *   既存のテストが引き続きパスすることを確認する。
    *   必要に応じて、新しいUI要素やロジックに対するテストを追加する。
3.  **PlaywrightによるE2Eテストの検討:**
    *   視覚的な変更がユーザー体験に影響を与えないか、E2Eテストで確認することを検討する（ただし、このフェーズでは実装しない）。
4.  **ビルドとデプロイの確認:**
    *   `npm run build` でビルドが成功することを確認する。
    *   `npm run dev` で開発サーバーが正常に起動し、変更が反映されていることを確認する。
